## Known limitations / wrinkles

1. **HttpUri parser patch.** Servarr serializes the health-check `wikiUrl` field as a string even though the spec models it as an object. `scripts/regen-clients.sh` includes a post-regen patch step. Re-running regen will re-apply the patch; the patched files carry the `ARRSTACK_HTTPURI_STR_OK` marker.
2. **Jellyfin first-run wizard.** The docker test container needs a one-time wizard pass before `library_search` works. v0.1.0 integration tests skip Jellyfin write paths; v0.2 should script the wizard.
3. **Prowlarr deferred.** Per CLAUDE.md hard rule #7, Prowlarr and `stack.*` cross-cutting tools land in v0.2.
4. **Per-user Jellyfin scoping.** Single-user only in v0.1.0; the `default_user_id` config field is honored, but the per-call `user=` override is deferred.
5. **No Unicode normalization table.** ibis-bot's MusicBrainz dash-replacement table was not ported. If Lidarr trips on exotic dashes again, port `_UNICODE_REPLACEMENTS` from ibis-bot's `tools/music.py`.
